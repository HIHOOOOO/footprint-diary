### GPS 실제 구현 (세션 2)

`addFootprint` 함수가 `MOCK_ADDRESSES` 배열에서 랜덤 주소를 고르는 가짜 구현이었다. `navigator.geolocation.getCurrentPosition`으로 교체하고, 위도/경도를 주소로 변환하는 역지오코딩이 필요했다.

카카오 지도 API는 API 키가 필요해서 패스. OpenStreetMap Nominatim API를 선택 — 무료, API 키 불필요, 한국어 응답 지원(`accept-language=ko`). 도로명이 있으면 `district + road` 조합으로, 없으면 city/county로 fallback하는 주소 파싱 로직 작성.

`granite.config.ts`의 `permissions` 타입은 `@apps-in-toss/plugins` 패키지 타입 정의를 직접 읽어서 확인. `GeolocationPermission = { name: 'geolocation', access: 'access' }` 형식으로 추가.

권한 거부, geolocation 미지원, 네트워크 오류 모두 `MOCK_ADDRESSES` fallback으로 처리해 앱이 죽지 않도록 함.

### 모바일 레이아웃 수정 (세션 2)

코드 리뷰에서 발견한 5가지 이슈를 한 번에 수정:

- `h-screen`(iOS Safari에서 주소창 포함 100vh로 레이아웃 잘림) → `style={{ height: '100dvh' }}`

`index.html`에 `viewport-fit=cover`, `index.css` body에 `env(safe-area-inset-*)` padding 추가해 safe area 활성화.

### 브라우저 흰 화면 원인 재진단

처음엔 Tailwind 미설치가 원인이라고 봤는데, 사용자가 "원인부터 재검토"를 요청해서 다시 파고들었다. 실제로는 두 가지가 겹쳐있었다:

1. **TDS Mobile ThemeProvider 미적용**: `@toss/tds-mobile`의 Button, TextField 등은 렌더 시 내부적으로 `useTheme()`을 호출한다. ThemeProvider 없이 쓰면 `"ThemeProvider로 감싸야 합니다."` 에러와 함께 React가 crash. `main.tsx`에서 브라우저 환경 시 `TDSMobileAITProvider` 대신 `TDSMobileProvider`(mock userAgent)로 감싸서 해결.

2. **Tailwind 미설치**: Tailwind가 없어도 흰 화면이 나오는 건 아니지만(`unstyled` 상태), 클래스가 무시되어 레이아웃이 깨진다. `@tailwindcss/vite` 플러그인 설치 후 `vite.config.ts`에 추가, `index.css` 맨 위에 `@import "tailwindcss"` 추가.

### 댓글 버튼 흰 화면 플래시

팝업에서 MessageCircle(댓글) 버튼 클릭 시 화면이 순간 하얗게 튀는 현상. 두 가지 수정을 적용했다:

1. **rAF 루프 throttle**: `setPopupPos`를 매 프레임 호출하지 않고 위치가 1px 이상 바뀔 때만 호출하도록 `lastPopupPosRef`로 비교. 불필요한 60fps React 리렌더 방지.

2. **AnimatePresence 제거**: 댓글 영역의 `motion.div height:'auto'` 애니메이션이 Framer Motion의 레이아웃 측정을 유발해 paint를 끊는 것으로 의심. CSS `max-height` 트랜지션으로 교체(TDS TextField/Button을 항상 마운트 상태로 유지, `showComments` 상태로 보여주고 숨김).

두 수정 후에도 사용자가 여전히 플래시 있다고 해서 원인이 확정되지 않은 상태로 세션 종료. 외부 팝업 wrapper의 `motion.div`가 원인일 수 있고, TDS Emotion FOUC일 가능성도 남아있다.

### localStorage 저장 기능 구현

앱을 껐다 켜면 발자국 데이터가 사라지는 문제. localStorage로 날짜별 저장 기능을 추가했다.

- `toDateKey(date)` — 날짜를 `"2026-03-02"` 형태 키로 변환
- `loadFromStorage(date)` — `"fp-2026-03-02"` 키로 읽기, 파싱 실패 시 null 반환
- `saveToStorage(date, footprints)` — footprints 변경될 때마다 자동 저장 (useEffect)
- 앱 시작 시 초기값을 localStorage에서 로드 (`useState(() => loadFromStorage(...) ?? [])`)
- 날짜 이동(← → 버튼, 달력) 시 해당 날짜 데이터 불러오기 + selectedId 초기화

### 토스 검수 기준 검토 및 대응

권한 거부 선택 시에도 앱이 정상 동작(MOCK_ADDRESSES fallback)하므로 "권한 거부 시 정상 동작" 항목도 통과.
